## Why this matters

Four small, verified hygiene gaps: (1) CI jobs are split between Node 20 and
Node 24 across workflows — a dep dropping Node 20 support would fail CI
inconsistently; (2) `npm audit` reports the postcss <8.5.10 XSS advisory via
next's transitive pin (GHSA-qx2v-qp2m-jg93) — npm `overrides` resolves it
without touching next; (3) `@emnapi/runtime` sits extraneous in
node_modules; (4) react/react-dom/@types patches are one `npm install`
behind. None is urgent alone; together they're 30 minutes of debt the next
audit shouldn't re-find.

## Steps

### Step 1: postcss override

Add to root `package.json`:

```json
  "overrides": {
    "postcss": "^8.5.10"
  }
```

Then `rtk npm install`.

**Verify**: `rtk npm audit --omit=dev` → the postcss advisory is GONE
(0 moderate or above). `rtk npm ls postcss` → every instance ≥ 8.5.10.

### Step 2: prune + patches

`rtk npm prune` then `rtk npm install` (pulls in-range patches).

**Verify**: `rtk npm ls --depth=0` → no "extraneous" lines;
`rtk npm outdated` → react/react-dom/@types rows show current == wanted.

### Step 3: unify CI node versions

In the three workflow files, change every `node-version:` value to `24`.

**Verify**: `rtk grep -rn "node-version" .github/workflows/` → all `24`.

### Step 4: full gates (both worlds)

**Verify**: root gates green AND backend gates green (commands table). The
postcss override must not change build output — if any test fails, see STOP.
